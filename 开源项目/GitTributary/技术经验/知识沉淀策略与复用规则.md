# 知识沉淀策略与复用规则

> 记录时间: 2026-07-06
> 目标: 把阅读 GitTributary 的经验稳定沉淀成可复用知识, 并让 Codex 后续自动沿用这套规则。

## 核心想法

这套知识库采用三层沉淀:

```text
开源项目/GitTributary/技术经验
  从具体项目出发, 记录项目导读经验

语言基础
  从项目中抽出语法、类型系统、语言机制

全栈工程
  从项目中抽出框架、架构、工程实践和技术模型
```

三者不是互相替代, 而是互相回链。

## 三类笔记的边界

### 1. 项目导读经验

路径:

```text
technical-note/开源项目/GitTributary/技术经验/
```

侧重点:

```text
项目为出发点到知识
```

适合记录:

```text
某个文件在 GitTributary 中的职责
某个用户动作如何穿过前端组件和后端命令
某个模块为什么这样组织
读这段项目代码时遇到哪些知识点
这些知识点如何服务于当前项目理解
```

例子:

```text
App.tsx 前端应用外壳解读
GitPanel 二级 Shell 解读
SitePanel 发布工作台状态流解读
Tauri invoke 在 GitTributary 中的调用链
```

### 2. 语言基础

路径:

```text
technical-note/语言基础/
```

侧重点:

```text
语法基础特性
```

适合记录:

```text
TypeScript import/export
interface 与 type
unknown / Partial / satisfies
JSX 基础语法
Rust Option / Result
Rust trait / module / ownership
```

写法要求:

```text
脱离具体项目, 用更小、更干净的例子讲清规则。
可以引用 GitTributary 作为来源案例, 但不要把它写成项目日志。
```

### 3. 全栈工程

路径:

```text
technical-note/全栈工程/
```

侧重点:

```text
技术框架的理解
```

适合记录:

```text
React 组件模型
React 状态和渲染机制
Vite 应用入口
Tauri 前后端桥接
桌面应用架构
插件式 Shell 设计
本地状态持久化
```

写法要求:

```text
重点讲框架心智模型、架构边界、工程权衡。
不要写成语法手册。
```

## 每次读代码时的工作流

以后每读一个文件或功能, 按这个节奏:

```text
1. 先写项目导读经验
   放到 开源项目/GitTributary/技术经验

2. 从项目导读里抽语言知识
   放到 语言基础

3. 从项目导读里抽工程知识
   放到 全栈工程

4. 三类笔记互相回链
```

比如阅读 `App.tsx`:

```text
项目导读经验:
开源项目/GitTributary/技术经验/App.tsx 前端应用外壳解读.md

语言基础:
语言基础/TypeScript/import 与 export.md
语言基础/TypeScript/interface 与 type.md

全栈工程:
全栈工程/React/React 应用入口与根组件.md
全栈工程/React/应用 Shell 与模块注册表.md
全栈工程/Tauri/Tauri invoke 前后端桥接.md
```

## 项目导读笔记模板

```md
# 文件名 / 功能名 解读

## 项目现场

这段代码在 GitTributary 中解决什么问题。

## 用户动作链路

用户做了什么, 代码如何响应。

## 关键代码

贴少量关键代码, 不整篇搬运。

## 当场需要补的知识

列出 React / TypeScript / Tauri / Rust 知识点。

## 可抽取的通用知识

哪些内容应该沉淀到语言基础或全栈工程。

## 回链

- 语言基础:
- 全栈工程:
```

## 抽取规则

判断一个知识点放哪里, 可以问三个问题:

```text
它离开 GitTributary 还成立吗?
```

如果不太成立, 放项目导读经验。

```text
它主要是在讲语法、类型、语言机制吗?
```

如果是, 放语言基础。

```text
它主要是在讲框架、架构、工程组织方式吗?
```

如果是, 放全栈工程。

## Codex 自动复用方式

当前项目根目录有一个本地 `AGENTS.md`。

它的作用:

```text
让 Codex 进入 GitTributary 工作区时,
自动知道这套三层知识沉淀策略。
```

为了不污染项目提交, `AGENTS.md` 已加入:

```text
.git/info/exclude
```

这意味着:

```text
AGENTS.md 是本机个人规则
不会进入 GitTributary 仓库提交
但本地 Codex 可以读取并复用
```

## 后续维护原则

```text
项目导读经验保留项目现场。
语言基础保持语法清晰。
全栈工程沉淀技术模型。
```

不要把所有东西都塞进一篇笔记。

更好的方式是:

```text
一篇项目导读经验
  -> 链接若干语言基础笔记
  -> 链接若干全栈工程笔记
```

这样知识会从项目现场自然长出, 但不会永远困在项目现场。
