# 开发能力提升计划与实验

**日期：** 2026-04-04
**目的：** 将能力清单转化为可执行的学习计划，通过实验（Lab）获得对技术栈的掌控力

---

## 设计原则

1. **每个 Lab 是独立可运行的小项目** — 不依赖 GitTributary 主项目
2. **每个 Lab 解决一个真问题** — 不是 "Hello World"，而是 GitTributary 中会遇到的具体场景
3. **Lab 之间有依赖链** — 前一个的产出是后一个的输入
4. **做完后有可复用的代码** — Lab 的核心代码可以直接搬到 GitTributary 中

### Lab 1.3：Trait 实现 Capability 接口

**目标：** 用 trait 抽象插件可用的系统能力

**做什么：**
- 定义 trait：

```rust
trait GitRead {
    fn status(&self, repo_path: &str) -> Result<GitStatus, PluginError>;
    fn log(&self, repo_path: &str, limit: usize) -> Result<Vec<GitCommit>, PluginError>;
    fn diff(&self, repo_path: &str, from: &str, to: &str) -> Result<String, PluginError>;
}

trait FileRead {
    fn read(&self, path: &str) -> Result<String, PluginError>;
    fn list_dir(&self, path: &str) -> Result<Vec<DirEntry>, PluginError>;
}
```

- 写两个实现：`MockGitRead`（返回假数据）和 `RealGitRead`（调用 `git2`）
- 写一个函数 `fn process_plugin<G: GitRead>(git: &G, manifest: &PluginManifest)`，接受任意实现
- 用 trait object `Box<dyn GitRead>` 做运行时多态

**验收标准：**
- 同一个 `process_plugin` 函数能接受 mock 和 real 两种实现
- 能用 `Box<dyn GitRead>` 存入 `HashMap<String, Box<dyn GitRead>>`

**涉及知识：**
- trait 定义与实现
- trait bound (`fn foo<T: GitRead>`)
- trait object (`dyn Trait`, `Box<dyn Trait>`)
- 泛型 vs trait object 的取舍

### Lab 4.2：Host Function — 宿主向 WASM 暴露能力

**目标：** 实现 Capability API 的雏形 — WASM 插件通过 host function 访问宿主资源

**做什么：**
- 宿主注册一个 host function `host_read_file`：

```rust
let host_read_file = Function::new(
    "host_read_file",
    [ValType::I64],
    [ValType::I64],
    UserData::default(),
    |plugin: CurrentPlugin, inputs: &[Val], outputs: &mut [Val], _user_data: UserData| {
        let file_path = plugin.memory_get_val(&inputs[0])?;
        let content = std::fs::read_to_string(file_path)?;
        plugin.memory_set_val(&mut outputs[0], &content)?;
        Ok(())
    },
);
```

- WASM 插件调用 `host_read_file` 读取文件内容，然后统计行数返回
- 宿主侧可以控制哪些路径允许访问（模拟权限校验）

**验收标准：**
- WASM 插件能通过 host function 读取到宿主文件系统中的文件
- 宿主拒绝访问白名单之外的路径时，WASM 插件收到错误

**涉及知识：**
- Extism host function 注册
- WASM 内存模型（宿主和 WASM 之间的数据传递）
- 权限校验的雏形

**时间：** 2-3 天

### Lab 6.3：权限系统雏形

**目标：** 实现 Capability 声明 + 运行时校验

**做什么：**

- 插件调用任何 host function 前，校验其 capabilities 是否包含所需权限

**涉及知识：**
- 权限模型设计
- 运行时拦截（Decorator / Middleware 模式）
