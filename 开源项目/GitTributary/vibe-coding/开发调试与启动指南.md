## 二、首次启动很慢是正常的

`npm run tauri dev` 第一次会从头编译整个 Rust 依赖树，耗时几分钟。

- **不要**因为「窗口迟迟没出现」就以为是 bug。
- **不要**在编译中途反复 启动 / 杀进程（这是踩坑根源，见第三节）。
- 依赖编译一次后会缓存，之后启动很快（`cargo check` 可秒级完成）。

---

## 三、实战踩坑：cargo 文件锁死锁导致窗口打不开

### 症状
- 跑了 `npm run tauri dev`，但桌面窗口一直不出现。
- 怀疑是代码实现有问题。

### 排查过程
1. 确认不是 Node 端问题：`npm run dev` 的 Vite（端口 1420）能正常起。
2. 跑 `cargo check`，日志只有一行：
   ```
   Blocking waiting for file lock on package cache
   ```
   说明 **没有编译错误**，而是卡在等待 cargo 的全局锁 `~/.cargo/.package-cache`。
3. 列出 cargo 进程：
   ```bash
   ps -ax -o pid,ppid,etime,command | grep -i cargo | grep -v grep
   ```
   发现多个孤儿进程（`PPID=1`）已经跑了好几分钟还没退出：
   ```
   78145  1  08:50  cargo run --no-default-features --color always --
   79784  1  07:13  cargo run --no-default-features --color always --
   82122 ...  05:57  cargo run --no-default-features --color always --
   ```
   外加两个 rust-analyzer 的 `cargo metadata` 也卡了 20+ 分钟。

### 根因
反复「启动 `tauri dev` → 直接 kill 父进程」后，底层的 Rust 编译进程（`cargo run`）变成**孤儿进程没有退出**，互相**死锁在 `~/.cargo/.package-cache` 这把全局锁**上。于是每次新启动一编译就卡在 "Blocking waiting for file lock"，窗口永远出不来。

**结论：不是代码实现的问题，是环境里堆积的僵死 cargo 进程占着锁。**

### 解决
强制清掉所有卡死的 cargo 进程，释放锁：
```bash
# 1. 找出卡死的 cargo 进程 PID
ps -ax -o pid,ppid,etime,command | grep -i cargo | grep -v grep

# 2. 杀掉这些孤儿 cargo 进程（替换成实际 PID）
kill -9 <pid1> <pid2> <pid3> ...

# 3. 确认锁已释放：cargo check 应秒级完成而非卡住
cd src-tauri && cargo check
```
清理后 `cargo check` 在 0.79s 完成 → 锁已释放，实现本身完全正常。

## 四、正确的停止方式（避免再造孤儿进程）

- 想停 `tauri dev`：在它运行的那个终端里**按一次 `Ctrl+C`**，让它优雅退出并带走子进程。
- **不要**直接 `kill -9` 父进程，否则底层 `cargo run` 会变成孤儿继续占锁。
- 万一又卡在 "Blocking waiting for file lock"，就是此问题复发，按第三节清理残留 cargo 进程即可。

## 五、清理端口与进程的常用命令（macOS）

```bash
# 查看 Vite 端口占用
lsof -ti tcp:1420
lsof -ti tcp:1421   # HMR

# 查看相关开发进程
ps -ax -o pid,ppid,command | grep -i -e vite -e tauri -e gittributary -e esbuild | grep -v grep

# 查看 cargo 进程（排查锁死）
ps -ax -o pid,ppid,etime,command | grep -i cargo | grep -v grep

# 查看 cargo 全局锁文件
ls -la ~/.cargo/.package-cache
```

> 注意：rust-analyzer（IDE 代码索引）也会跑 `cargo metadata`，正常情况别误杀；
> 但若它已卡死很久并占着锁，可以杀掉，rust-analyzer 会自动重启它。

## 六、断点调试

- **前端**：在 Tauri 窗口里右键 → Inspect Element 打开 WebView DevTools；或用 `npm run dev` 在 Chrome 里调，断点打在 `.ts/.tsx`。
- **Rust 端**：VS Code 装 `rust-analyzer` + `CodeLLDB`，对 `src-tauri` 配 `launch.json` 进行断点调试。

## 七、一句话总结

> 桌面窗口打不开，**先别怀疑代码**。多半是首次编译慢，或反复杀进程留下的孤儿 cargo 死锁在 `~/.cargo/.package-cache` 上。清掉僵死 cargo 进程，用 `Ctrl+C` 优雅退出，问题即解。
