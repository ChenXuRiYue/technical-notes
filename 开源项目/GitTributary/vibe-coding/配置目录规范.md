# `.git-tributary/` 配置目录规范

> 类似于 `~/.claude`、`~/.codex`、`~/.config/git` 等，
> `~/.git-tributary/` 是 GitTributary 应用的**全局配置根目录**。
> 所有持久化数据(凭证、工作区状态、配置、同步缓存)统一在此管理。

---

## 一、位置

```
~/.git-tributary/
```

- macOS: `/Users/<user>/.git-tributary/`
- Linux: `/home/<user>/.git-tributary/`
- Windows: `C:\Users\<user>\.git-tributary\`

应用首次启动时自动创建。

---

## 二、完整目录结构

```
~/.git-tributary/
├── config.json                 ← 应用级元配置(数据中心自身行为)
│
├── data/                       ← 数据中心 KV 存储(JSONL)
│   ├── settings.jsonl          ← 用户设置(主题/语言/备份策略...)   [public]
│   ├── workspace.jsonl         ← 工作区(当前仓库/最近仓库/设备信息) [public]
│   ├── ui-state.jsonl          ← UI 布局偏好(宽度/折叠/视图模式)   [public]
│   ├── review.jsonl            ← 复习曲线数据                      [public]
│   ├── tags.jsonl              ← 笔记标签索引                      [public]
│   └── plugins/                ← 各插件私有配置
│       └── <plugin-id>.jsonl                                        [public]
│
├── credentials/                ← 凭证与密钥(仅本地,永不同步)
│   ├── github.json             ← GitHub 凭证(PAT / OAuth token)    [private]
│   ├── ssh.json                ← SSH 密钥路径 + passphrase 引用     [private]
│   └── keys/                   ← 应用生成的密钥对
│       ├── sign.pub            ← 公钥(可同步,用于验证)            [public]
│       └── sign.key            ← 私钥(仅本地)                     [private]
│
├── profiles/                   ← 配置档(多套配置快照)
│   ├── _active.json            ← 当前激活的 profile 指针            [public]
│   ├── work.jsonl              ← "工作" 配置档                      [public]
│   └── personal.jsonl          ← "个人" 配置档                      [public]
│
├── cache/                      ← 可丢弃的缓存(不同步,丢了可重建)
│   ├── git-log-index.json      ← git log 索引缓存
│   ├── file-meta.json          ← 文件元数据缓存
│   └── ai-embeddings/          ← AI 向量缓存
│
└── sync/                       ← 远程同步状态
    ├── remote.json             ← 同步远程仓库配置(URL/分支)        [public]
    ├── last-sync.json          ← 上次同步时间戳 + commit SHA
    └── conflicts/              ← 同步冲突暂存
```

---

## 三、模块职责

| 子目录 | 职责 | 可见性 | 同步到远程? |
| --- | --- | --- | --- |
| `data/` | 所有配置项的 KV 存储 | Public | ✅ 随 Git push 同步 |
| `credentials/` | 认证凭证、密钥 | **Private** | ❌ 永不离开本机 |
| `profiles/` | 多套配置快照 | Public | ✅ 可同步 |
| `cache/` | 运行时缓存 | — | ❌ 不同步,可丢弃 |
| `sync/` | 同步引擎状态 | Mixed | 部分同步(remote.json 可以) |

## 四、凭证管理(`credentials/`)

### GitHub 凭证

```json
// credentials/github.json
{
  "username": "ChenXuRiYue",
  "token": "<encrypted PAT>",
  "token_type": "pat",           // "pat" | "oauth" | "ssh"
  "created_at": "2026-06-24T00:00:00Z"
}
```

- token 字段使用本地 AES 加密存储(密钥来自系统钥匙串或主密码)
- 永远不进 git,不同步

### SSH 配置

```json
// credentials/ssh.json
{
  "default_key": "~/.ssh/id_ed25519",
  "keys": [
    {
      "name": "personal",
      "path": "~/.ssh/id_ed25519",
      "has_passphrase": true
    },
    {
      "name": "work",
      "path": "~/.ssh/work_rsa",
      "has_passphrase": false
    }
  ]
}
```

- 只存路径引用,不复制私钥文件本身
- passphrase 如需缓存,走系统钥匙串

## 五、workspace 命名空间(`data/workspace.jsonl`)

记录当前工作状态:

| Key | 含义 | 示例值 |
| --- | --- | --- |
| `device.id` | 设备唯一标识(hostname+user hash 前 8 位) | `"a3f2c1d8"` |
| `device.name` | 设备友好名 | `"MIdeMacBook-Pro"` |
| `repo.active` | 当前聚焦的 Git 仓库路径 | `"/Users/mi/note/technical-notes"` |
| `repo.recent` | 最近打开的仓库列表(max 10,最新在前) | `["/Users/mi/note/...", ...]` |

## 六、同步机制(`sync/`)

### 数据同步流程

```
本地 data/ + profiles/
     ↓ git add (只 public 文件)
     ↓ git commit -m "sync: <device_id> <timestamp>"
     ↓ git push → 远程仓库
     
远程仓库
     ↓ git pull
     ↓ 合并(同 key 取 timestamp 最大的)
本地 data/ 更新
```

### 远程仓库配置

```json
// sync/remote.json
{
  "url": "git@github.com:ChenXuRiYue/gt-config.git",
  "branch": "main",
  "auto_sync": true,
  "interval_seconds": 300
}
```

用户可以用一个专门的 private GitHub repo 存配置(类似 dotfiles 仓库)。

### 同步范围

- `data/*.jsonl`(Public 命名空间)→ 同步
- `profiles/*.jsonl` + `profiles/_active.json` → 同步
- `credentials/` → **永不同步**
- `cache/` → **不同步**(重建代价低)

## 七、与 .gitignore 的关系

如果 `.git-tributary/` 放在某个 Git 仓库内部(内嵌模式),需要在该仓库的 `.gitignore` 中忽略:

```gitignore
.git-tributary/
```

但当前默认设计是放在 `~/`(全局),不在任何笔记仓库内部,所以不需要 gitignore。

## 九、命名说明

目录名用 `.git-tributary`(带连字符),因为:
- 与项目名 GitTributary 对应
- `.` 前缀表示隐藏/配置目录(Unix 惯例)
- 连字符风格与 `.git`、`.ssh`、`.config` 一致
- 不与现有 `.git` 目录冲突

## 十、访问控制规则(强制)

**数据中心(`gt-store`)是 `~/.git-tributary/` 的唯一读写入口。**

### 敏感数据安全级别

| 级别 | 含义 | 展示规则 | 示例 |
| --- | --- | --- | --- |
| **L0** | 绝对机密 | 完全掩码(`••••••••`),不露任何原始字符 | access_token、私钥、passphrase |
| **L1** | 敏感 | 可部分展示(如路径只显示文件名) | SSH 密钥路径 |
| **L2** | 普通 private | 正常展示,但仅本地存储 | 设备名、设备 ID |

**L0 规则(绝对):**
- 前端展示: `••••••••`,不允许显示任何原始字符(包括前 N 位)
- 日志/错误信息: 禁止出现明文
- Tauri command 返回: 只返回 `has_token: bool`,不返回值本身
- 只有内部 `get_git_token_raw()` 可获取明文(用于 push/pull 认证)

| 规则 | 说明 |
| --- | --- |
| 独占读写 | 任何模块(gt-git / AI / 插件 / 前端)不得直接操作 `~/.git-tributary/` 下的文件,必须通过 `gt-store` 的 API |
| 包括凭证 | `credentials/` 的读取也由 gt-store(未来的 `gt-store::credentials` 子模块)统一提供,其他模块调接口获取 |
| 包括同步 | 云端 GitHub 配置仓库的 pull/push/merge 由 gt-store 的 sync 模块负责,其他模块只消费最终合并后的数据 |
| 包括缓存 | `cache/` 的写入也走 gt-store,即使是 gt-git 的 log 索引缓存,也由 gt-store 分配路径并管理生命周期 |

**为什么:**
- 单一入口 → 不会出现多个模块同时写同一文件导致数据损坏
- 统一的 public/private 分类 → 不可能因为某个插件直接读 credentials/ 就绕过了可见性控制
- 统一的同步语义 → 不会出现"本地改了但没同步"或"同步了不该同步的"

**违反此规则的代码视为 bug,review 时直接打回。**

---

## 十一、一句话总结

> `~/.git-tributary/` = GitTributary 的大脑:凭证在 credentials/(永不外泄),配置在 data/(Git 同步),缓存在 cache/(可丢弃),多设备靠 sync/ 协调。
