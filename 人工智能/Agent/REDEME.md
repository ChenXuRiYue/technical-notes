# 📌 Agent

## 📄 是什么

**Agent（智能体）** 是一个以 LLM 为核心的自主系统，能够感知环境、制定计划、调用工具、执行任务并自我反思。

```
Agent = LLM（大脑） + Tools（手脚） + Memory（记忆） + Planning（规划）
```

**LLM 是 Agent 的一个组件，不是 Agent 本身。** LLM 负责推理和生成，Agent 负责把推理结果转化为实际动作。

## 📄 Agent vs LLM

| 维度 | LLM | Agent |
|------|-----|-------|
| 本质 | 文本生成模型 | 自主执行系统 |
| 能力 | 理解 + 生成文本 | 理解 + 规划 + 调用工具 + 执行 + 反思 |
| 交互 | 一问一答 | 多步自主完成任务 |
| 状态 | 无状态（每次对话独立） | 有状态（记住上下文和中间结果） |
| 工具 | 无 | 可调用外部工具（API、数据库、代码执行等） |
| 例子 | ChatGPT 对话 | Claude Code、AutoGPT、Devin |

## 📄 发展阶段

### 前 Agent 时代（2022 年以前）

LLM 只能做文本生成，没有工具调用、没有记忆、没有规划能力。"Agent"这个概念在这个阶段还停留在学术论文中。

| 时间 | 事件 | 意义 |
|------|------|------|
| 2017 | Transformer 论文 | LLM 的架构基础 |
| 2018 | GPT-1 / BERT | 预训练语言模型出现 |
| 2020 | GPT-3 | 展示了 few-shot 学习能力，但仍是纯文本生成 |
| 2021 | LoRA 论文发布 | 低秩适应微调，大幅降低模型定制成本，让个人和小团队也能微调大模型 |

### 萌芽期（2022-2023）— 从"能想"到"能做"

这个阶段的关键突破是 **让 LLM 学会使用工具和思考**。

| 时间 | 事件 | 意义 |
|------|------|------|
| 2022.01 | Chain-of-Thought（CoT） | 让 LLM 学会"分步思考"，推理能力大幅提升 |
| 2022.03 | ChatGPT（GPT-3.5） | LLM 进入大众视野，对话能力质变 |
| 2022.10 | ReAct 论文 | 提出 Reasoning + Acting 循环，Agent 的理论基础 |
| 2023.03 | GPT-4 + Function Calling | LLM 第一次能结构化地调用外部工具，Agent 的工程基础 |
| 2023.03 | AutoGPT 爆发 | 第一个"出圈"的 Agent 项目，展示了 LLM 自主完成任务的可能性 |
| 2023.04 | LangChain 热潮 | Agent 开发框架标准化，工具链生态开始形成 |

### RAG 热潮（2023-2024）— 让 LLM "看见"外部知识

RAG（Retrieval-Augmented Generation，检索增强生成）解决了 LLM 的知识局限：

- **问题：** LLM 的知识截止到训练数据，无法获取实时信息，且会产生幻觉
- **方案：** 先检索相关文档，再让 LLM 基于检索结果生成回答

```
用户问题 → 检索相关文档 → 文档 + 问题一起喂给 LLM → 生成回答
```

| 时间 | 事件 | 意义 |
|------|------|------|
| 2023 | RAG 范式确立 | LLM + 外部知识库，幻觉大幅减少 |
| 2023 | 向量数据库热潮 | Pinecone、Milvus、Weaviate，RAG 的基础设施 |
| 2024 | RAG 工程化 | 从"能用"到"好用"，分块策略、重排序、混合检索等优化 |

**RAG vs 微调（LoRA）：**

| 维度 | RAG | 微调（LoRA） |
|------|-----|-------------|
| 原理 | 检索外部知识注入上下文 | 修改模型参数，让模型"学会"新知识 |
| 成本 | 低（只需建索引） | 中（需要训练数据和算力） |
| 实时性 | 高（知识库可随时更新） | 低（需要重新训练） |
| 适用场景 | 知识问答、文档搜索 | 特定领域风格、格式定制 |
| 幻觉 | 仍有，但减少 | 仍有，但减少 |

### Agent 工程化（2024-2025）— 从 Demo 到产品

| 时间 | 事件 | 意义 |
|------|------|------|
| 2024.01 | MCP 协议发布 | Anthropic 提出，统一 LLM 和外部工具的通信标准 |
| 2024 | Claude 3.5 Sonnet | 推理能力质变，Agent 的"大脑"升级 |
| 2024 | Multi-Agent 框架 | CrewAI、AutoGen 等，多 Agent 协作成为主流 |
| 2025 | Claude Code / Cursor | Agent 进入日常开发工作流，不再是玩具 |
| 2025 | Agent 平台化 | Dify、Coze 等低代码平台，非技术人员也能搭建 Agent |

### Agent 大众化（2026）— OpenClaw 引爆

| 时间 | 事件 | 意义 |
|------|------|------|
| 2026.02 | OpenClaw 开源发布 | 首日 9,000+ Stars，一周突破 150,000+ Stars |
| 2026.02 | OpenClaw 生态爆发 | 135,000+ 实例部署，全球社区迅速建立 |
| 2026.03 | Agent 进入普通用户日常 | 非技术人员通过聊天界面使用 Agent 完成日常任务 |

**OpenClaw 的定位：** 个人 AI 助手，真正能"做事"的 Agent — 清理邮箱、发邮件、管理日历、航班值机等，通过 WhatsApp、Telegram 等聊天界面操作。

**OpenClaw 为什么重要：**

| 维度 | 说明 |
|------|------|
| 技术栈 | OpenClaw + MCP + Skill，验证了 Agent 技术栈的成熟 |
| 用户门槛 | 零代码，通过聊天即可使用，Agent 第一次真正走进大众 |
| 行业影响 | IDC 预测全球活跃 Agent 从 2025 年 2,860 万增长到 2030 年 22.16 亿（CAGR 139%） |
| 安全挑战 | 上线不到两个月已有 135,000+ 实例暴露在公网，安全风险凸显 |

**OpenClaw 之于 Agent，就像 ChatGPT 之于 LLM — 第一次让大众感受到 AI Agent 的实际价值。**

### 发展脉络总结

```
纯文本生成（GPT-3）
  → 学会思考（CoT）
    → 学会使用工具（Function Calling）
      → 学会获取外部知识（RAG）
        → 学会自主规划（ReAct / Plan-Execute）
          → 学会协作（Multi-Agent）
            → 进入开发者工作流（Claude Code / Cursor）
              → 走进大众（OpenClaw）
```

**一句话：** Agent 的发展就是让 LLM 从"只能想"到"能想能做"的过程。

## 📄 核心组成

### 大脑（LLM）

负责推理、决策、自然语言理解与生成。Agent 的能力上限取决于 LLM 的能力。

### 工具（Tools）

LLM 本身只能生成文本，工具让它能做事情：

| 工具类型 | 例子 |
|----------|------|
| 代码执行 | 运行 Python/Shell 脚本 |
| 文件操作 | 读写文件、搜索代码 |
| API 调用 | 搜索引擎、数据库、第三方服务 |
| 浏览器 | 网页访问、表单填写 |

工具通过 **MCP（Model Context Protocol）** 等协议统一接入，让 LLM 以标准化方式发现和调用外部能力。

### 记忆（Memory）

| 类型 | 说明 | 例子 |
|------|------|------|
| 短期记忆 | 当前对话的上下文 | 对话历史 |
| 长期记忆 | 跨对话持久化的知识 | 用户偏好、项目规范 |
| 工作记忆 | 当前任务的中间状态 | 已完成的子任务、待处理列表 |

### 规划（Planning）

Agent 面对复杂任务时的拆解和执行策略：

| 模式 | 说明 |
|------|------|
| ReAct | 思考 → 行动 → 观察 → 循环 |
| Plan-Execute | 先制定计划，再逐步执行 |
| Reflection | 执行后自我反思，调整策略 |
| Multi-Agent | 多个 Agent 协作，分工完成 |

## 📄 从 LLM 到 Agent 的工程投入

| 方向 | 工作内容 |
|------|----------|
| 工具集成 | 封装 MCP、安全系统，产出可插拔、可观测、安全的工具平台 |
| 状态管理 | 任务分解、持久化中间状态、中断恢复、多 Agent 协作 |
| 可靠性 | 错误分类与处理、自我反思、人工干预接口、超时限制 |
| 可观测性 | 全链路日志、性能指标、调试面板、A/B 测试 |
| 安全合规 | 最小权限、输入输出过滤、操作审计、沙箱执行 |

## 📄 Agent 产品生态

| 产品 | 类型 | 特点 |
|------|------|------|
| Claude Code | 终端 Agent | MCP 扩展、自主任务、Git 深度集成 |
| Cursor | AI IDE | Composer、可视化 diff、多模型 |
| Devin | 自主 Agent | 端到端完成开发任务 |
| AutoGPT | 开源 Agent | 自主循环、任务分解 |
| Dify | Agent 平台 | 低代码搭建 Agent 工作流 |
| LangChain | Agent 框架 | Python/JS，工具链编排 |

## 📄 学习路线

```
1. 理解 LLM 基础 → token、prompt、上下文窗口
2. 理解工具调用 → Function Calling → MCP 协议
3. 理解 Agent 架构 → ReAct、Plan-Execute、Reflection
4. 动手实践 → Claude Code / Dify / LangChain 搭建 Agent
5. 深入 → Multi-Agent 协作、状态管理、安全合规
```

## 🌳 生长思考

- **Agent 是 LLM 的工程化落地** — LLM 解决"想"的问题，Agent 解决"做"的问题
- **工具是 Agent 的边界** — Agent 能做什么，取决于它能调用什么工具
- **记忆是 Agent 的灵魂** — 没有记忆的 Agent 只是一个高级的问答机器人

## 💭 反复绊脚

- Agent 不是更聪明的 LLM，而是一个能做事的系统
- MCP 不是模型本身，而是 LLM 和外部工具之间的中间件
- Multi-Agent 不是多个 LLM 对话，而是多个独立 Agent 各自调用工具协作完成任务
