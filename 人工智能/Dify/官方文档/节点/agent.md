# 📌 Agent
## 🔖 概述
Agent 代理节点可以让 LLM 自主控制工具，使其迭代决定使用哪些工具以及如何使用它们。

## 🔖 代理策略

智能代理策略定义了智能代理**如何思考和行动**。

### 📝 函数调用

使用大型语言模型的原生函数调用能力，通过工具参数直接传递工具定义。大型语言模型使用其内置机制决定何时以及如何调用工具。

1. 定义工具（JSON 对象或者某种泛式描述工具）
2. 调用模型（传递定义，上下文 prompt）
3. 模型响应（响应可以调用工具的入参）
4. 执行、循环（大模型重试 + 试错调整）

### 📝 推理和行动

使用结构化提示词来引导大型语言模型通过明确的推理步骤。遵循**思维 → 行动 → 观察**循环进行透明的决策制定。

#### React 模式 （R eason + Act）

推理和行动交织在一起，形成一个闭环。

1. 模型被提示遵循 `Thought -> Action -> Observation` 的循环。
2. **Thought（思考）**：模型先输出一段文字，分析当前情况，规划下一步。
3. **Action（行动）**：模型决定调用哪个工具。
4. **Observation（观察）**：系统执行工具并返回结果，模型看到结果后再次思考。

## 🔖 配置

- 模型选择
  选择强大的模型。如果使用函数调用，确保模型支持函数调用。

  > 区分 Prompt 函数调用和 原生函数调用
  > 原生函数调用：
  >
  > - 结构化输出强制约束
  > - 工具列表作为上下文输入
  > - 返回类型不是纯文本，而是结构化消息

- 工具配置

  - 授权
  - 描述
  - 参数

- 指令和上下文

- 执行控制

  - 最大迭代次数
  - 记忆

- 工具参数自动生成



## 🔖 最佳实践

- **清晰的工具描述**帮助智能代理了解何时以及如何有效使用每个工具。
- **适当的迭代限制**防止失控成本，同时为复杂任务提供足够的灵活性。
- **详细的指令**提供关于智能代理角色、目标以及任何约束或偏好的上下文。
- **记忆管理**根据你的用例要求平衡上下文保留与标记效率。
