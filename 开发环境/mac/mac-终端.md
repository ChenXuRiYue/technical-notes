## 🔖 权限控制

⚙️ 超级管理员权限

最方便，但不一定安全。要确保掌控权在自己手上

| 命令          | 作用                                   | 适用场景                                               |
| ------------- | -------------------------------------- | ------------------------------------------------------ |
| `sudo <命令>` | 以 root 权限执行单条命令               | 最常用的方式，用完即走，安全                           |
| `sudo -i`     | 切换为 root 用户并加载 root 的环境变量 | 需要连续执行多条管理命令时使用                         |
| `sudo su`     | 切换为 root 用户（不加载环境变量）     | 类似于 `sudo -i`，但环境配置可能不同                   |
| `sudo -k`     | 清除缓存的凭证                         | 操作完成后，强制下次使用 `sudo` 时重新输入密码，更安全 |
| `sudo -v`     | 更新凭证缓存时间                       | 延长免密时间，无需重新输入密码                         |

时机：

- **修改系统配置文件**：例如修改 `/etc/hosts`、`/etc/profile` 等。
- **安装全局软件**：使用某些包管理器安装软件到系统目录时。
- **管理后台服务**：启动、停止或重启系统服务（如 `launchctl` 相关操作）。
- **操作受保护目录**：向 `/usr/local/bin`、`/Library` 等目录写入文件。
- **更改文件所有权**：使用 `chown` 或 `chmod` 修改不属于你的文件。



## 🔖 进程管理

⚙️ 关闭端口应用程序

- `kill -9 $(lsof -ti :8080)`
  - `kill -9` 强制杀死
  - `$()` 占位符
  - `lsof -ti :8080` 查找占用 8080 端口号的 pid （process id 进程 ID）
    - `-t` 仅见模式 `(Terse mode)`
    - `-i` 表示查看网络接口 `Internet`
- `lsof -ti :8080 | xargs kill -9`
  - `|` 管道符
    - 将前一个命令的标准输出，直接变为后一个命令的标准输入
  - `xargs kill -9`
    - `xargs` 从标准输入构建并执行命令。它读取管道传来的内容，将其作为参数追加到后面的命令
