| -------------- | ---- | ---- |
| 入职、更换电脑 | 2    |      |
|                |      |      |

🔗 引用

进入公司后的首要任务是完成 Git 环境的初始化配置。Git 作为分布式版本控制系统，安装后可实现本地仓库管理，并与 GitHub 等代码托管平台协同开发。对于安全性要求较高的代码仓库，需通过身份验证机制确保操作权限 —— 用户需在本地生成 SSH 密钥对，并将公钥添加至平台的密钥管理系统中。通过这种基于密钥的身份验证方式，可实现安全、免密的代码仓库访问与协作。

## 🔖 基于旧tag，新开发分支发版本。

⚙️ 

```txt
# 拉取最新的 tag
git fetch --all --tag

# 基于线上 tag v1.2.3 创建并切换到新的分支
git checkout -b hotfix/v1.2.4 v1.2.3

# 合并开发分支代码
git merge develop

# 推送到远程分支
```

## 🔖 取消文件跟踪 git rm --cached

⚙️ 

```txt
# 取消单个文件的跟踪（保留在工作区，仅从暂存区移除）
git rm --cached <file>

# 取消整个目录的跟踪
git rm --cached -r <directory>

# 取消跟踪后，将文件加入 .gitignore
echo "<file>" >> .gitignore
git add .gitignore
git commit -m "stop tracking <file>"
```

🪏 **经验**

| 场景 | 使用 | 频率 |
| ---- | ---- | ---- |
| 误提交了配置文件（如 application.yml、.env），需要取消跟踪但保留本地文件 | 1 | |
| 初始化项目时忘记添加 .gitignore，后续补救 | | |

🔗 引用

`--cached` 的含义是只操作暂存区（index），不影响工作区的物理文件。如果不用 `--cached`，`git rm` 会同时删除工作区文件。

## 🔖 暂存工作区修改 git stash

⚙️ 

```txt
# 暂存当前工作区和暂存区的修改
git stash

# 暂存时添加备注信息
git stash push -m "描述信息"

# 查看所有 stash 记录
git stash list

# 恢复最近一次 stash（保留 stash 记录）
git stash apply

# 恢复最近一次 stash（同时删除 stash 记录）
git stash pop

# 恢复指定 stash
git stash apply stash@{n}

# 删除指定 stash 记录
git stash drop stash@{n}

# 清空所有 stash 记录
git stash clear
```

🪏 **经验**

| 场景 | 使用 | 频率 |
| ---- | ---- | ---- |
| 开发到一半需要切分支处理紧急 bug，先暂存当前修改 | 1 | 2 |
| pull 远程代码前，先暂存本地未完成的修改 | 1 | |
| 尝试性修改不想提交，临时保存 | | |

🔗 引用

`git stash` 默认不会暂存未跟踪（untracked）的文件。如需包含，使用 `git stash -u`。
