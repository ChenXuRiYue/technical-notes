# technical-notes

使用 Typora + markdown 的方式来做一些相对散乱的笔记是我大二开始的习惯。
在大学校招期间，我尝试过飞书、语雀、wps 等。这些软件都很棒，有优秀的UI，安全的云但是它们都太重了。切入笔记状态路径较长。

2024-9-15，平常的一天。
我将大学时期的算法笔记提交到 github ，命名为 college-years-algorithm-note。从此不再维护
我惊奇地发现 

- typora = 简洁UI的笔记工具

- github  = 云、git = 历史跟踪
- markdown 具有强大的表达能力

精心设计地 git comit、git push 是很令人着迷的。对我来说就像搭好积木拿相机拍拍，起个好名字。这是具有浓烈仪式感的过程

## 📌 为什么？

我从小就喜欢写东西，写日记、写笔记、写沉思录。十几本笔记本落在房间中。我和很多个人聊过这件事（主要是在这个 github）的意义。

1. 一个前辈说，他年轻的时候也写过，后来他没接受到正反馈，觉得有种自嗨感。
2. 有 AI ，近全部问题可以快速找到一个答案，为什么要麻烦的写一个笔记呢

我整理过自己的想法。它是迭代的：

>v2：
>
>1. 写笔记本身就是一种创造过程，本质和打游戏一样。也比生活中大多琐事有趣
>2. 我喜欢出整洁的笔记，好比把房间清理得更简洁。把乱的东西变干净，让我有种开心的感觉
>3. AI 对于检索是更高效的浏览器，很大部分它们的子问题是相同的。有浏览器了为什么做笔记？
>   - 这些工具带来的是短时记忆是不长久的，下次遇到还是困惑走一遍工作流
>   - 我必须避免幻觉带来的精力干扰，做笔记本身是记录遇到过的东西，很快我就会将工作遇到的大多数频繁遇见的问题积累下来。避免了多次检索，询问AI打断工作流
>4. 个人的记忆能力是有限的，长时间的记忆习惯积累，打造自己的记忆宫殿
>5. 做伟大有趣的事，总是是长时间，连续的。笔记是不终结的思考。它保留上下文，方便作者用更长的时间去感受，去发散，去解决。（电子笔记可重构，复原，留痕，便携）
