# technical-notes

使用 Typora + markdown 的方式来做一些相对散乱的笔记是我大二开始的习惯。
在大学校招期间，我尝试过飞书、语雀、wps 等。这些软件都很棒，有优秀的UI，安全的云，但是它们都太重了。切入笔记状态路径较长。

2024-9-15，平常的一天。
我将大学时期的算法笔记提交到 github ，命名为 college-years-algorithm-note。从此不再维护
我惊奇地发现 

- typora = 简洁UI的笔记工具

- github  = 云、git = 历史跟踪
- markdown 具有强大的表达能力

精心设计的 git commit、git push 是很令人着迷的。对我来说就像搭好积木拿相机拍拍，起个好名字。这是具有浓烈仪式感的过程

## 📌 为什么？

我从小就喜欢写东西，写日记、写笔记、写沉思录。十几本笔记本落在房间中。我和很多个人聊过这件事（主要是在这个 github）的意义。

1. 一个前辈说，他年轻的时候也写过，后来他没得到正反馈，觉得有种自嗨感。
2. 有 AI ，近全部问题可以快速找到一个答案，为什么要麻烦的写一个笔记呢

我整理过自己的想法。它是迭代的：

>v3：2026 6.1
>
>今天喝了一杯超大杯（约700毫升）的瑞幸生椰拿铁。思维变得非常活跃，蹦出了很多新想法。
>
>
>我的顶层目录中一直维护着一个名为 `temp.md` 的文件，这半年来它主要充当草稿箱的角色。过去处理任务时，我“做完即删”。
>
>我懒得删了，让 claude  扫描 Git 历史记录，识别哪些任务已完成，帮我把剩下任务按照重要性、紧急度和难度进行排序。
>
>AI 不仅清理了任务，还优化了排版——用优雅的完成符号和删除线来标记历史。在调整 Cloud Markdown 配置文件的过程中，我想起大学期间，我每周会写一些算法周报，记录我这周解决了什么难度的问题，学习了什么算法。我每周会将它们打印复习。这种可视化让我分泌多巴胺，更有成就感。
>
>
>我决定将这些记录保存下来。将完成的任务提到文末，用优雅的符号表达。将这些配置放在 claude 配置目录中。
>
>我越琢磨越起劲。为什么不能引入一个 积分系统呢？既然 AI 以这种程度参与了我的笔记系统。它已经作为了我笔记仓库的图书管理员，一个博学的教授。那么为什么不能作为一个游戏制度裁判长呢？我可以知道哪些事情是重要的，更好的感受进度并且得到正反馈。
>
>我小时候显然有过这种尝试，但是因为这套制度需要大量笔墨来维护，花出的精力甚至超过解决任务本身。但今年 openclaw 出世，AI 能力提升。世界已经不一样了。Agent 来做这个裁判长，它是最公正的维护者。我只需要专注享受一场又一场有趣的游戏，
>
>我开始考虑 这个游戏机制的评分模型、激励制度。怎么评价一个任务重要？这次任务的产出是否丰富充分？我是否真正完成了一件困难但正确的事情？这个调整模型过程引发了我的创造欲。它的问答记录是有趣的，我已经记录在笔记仓库中了。未来我可能会删除，但它会留在我的笔记仓库中，起码在 git  的 History 中。
>
>如果一切顺利，在不久的将来，这套机制会单独形成一个 Agent 会封装到我的 GitTributary 中。这是多么深刻的一次发散，GitTributary，统管我的多个笔记仓库。包括我的生活上的记录，技术事业上的记录，做一个统筹。所谓游戏人生。我将获得更多的里程碑，实现人生意义的时刻。这套信息也将内聚在一起，方便 Agent 回顾我的成长，像软件做报告一样，为我生产一个年报，丰富的游戏化的年报。
>
>
>进一步发散，Agent 真正来了。AI Coding 让我加快了我的编码速度，一两天完成一个中大型需求。让我不再痛苦地抓取、分析信息。注重灵感，思维上的前进。让我这小小的记忆海马体（记忆能力）的劣势淡化。
>
>- 我原来连简单的任务都不希望记录。比如记录一个工作流，一个小小的工作卡点。因为它们总是会耗一定时间。现在有了 AI agent 这个问题解决了。
>  
>
>- 我原来希望和我第一次实习一样，仅仅使用一个简单的工作流将 简历.pdf 中的内容抠出来，形成 markdown 做分析总结。当初只能用 gpt 4.0 效率甚至没多好。如今又大不相同了。我只需要在 .claude 中做一个配置。claude 在启动时读取一遍 prompt。于是它成了我的 导师、辩论者、图书管理员、陪伴者、回顾我一年工作的守望者、游戏机制个人荣誉的铭记者。
>  
>
>- 这是一个灵感至上的时代。而灵感总是来源于积累，对概念信息的深刻把握。不是简单问一句 AI：请告诉我这些信息，简单看一眼陌生的信息，接着问：请把握重点直到简短的变形的几句话。而是平时注意维护积累（个人信息系统），从而对各类信息更敏感，更能畅通无阻的把握，发散。如果这类信息刚好命中我的缓存，那我将会更加优雅的解决问题
>  
>
>- 当小孩痛哭，我会在旁边给糖果、播视频、打游戏各种方式去哄。直到最终会找到一种方式让他乖下来。为什么我不能运气很好的，第一次就拿出颗糖果哄好孩子呢？同理，我哥还有朋友们曾经和我多次倾诉他们的痛苦，暂时的他们没法解决生活上的问题。作为旁外人只能手忙脚乱搬出对人生的粗浅理解，试图缓解他们的困境。如果把难受的点记录下来，一时解决不了也没事。我相信总有推进。旁人不需要再艰难地代入猜测、更好的解决问题。何况还有 AI，2026.6.1 已经这样强大的 AI、未来的 AI。
>
>
>  同理，当我提供了足够多的信息（从我的技术理念笔记、生活笔记中明确提取），AI 将会帮助我。



>v2：
>
>1. 写笔记本身就是一种创造过程，本质和打游戏一样。也比生活中大多琐事有趣
>2. 我喜欢出整洁的笔记，好比把房间清理得更简洁。把乱的东西变干净，让我有种开心的感觉
>3. AI 对于检索是更高效的浏览器，很大部分它们的子问题是相同的。有浏览器了为什么做笔记？
>   - 这些工具带来的是短时记忆是不长久的，下次遇到还是困惑走一遍工作流
>   - 我必须避免幻觉带来的精力干扰，做笔记本身是记录遇到过的东西，很快我就会将工作遇到的大多数频繁遇见的问题积累下来。避免了多次检索，询问AI打断工作流
>4. 个人的记忆能力是有限的，长时间的记忆习惯积累，打造自己的记忆宫殿
>5. 做伟大有趣的事，总是是长时间，连续的。笔记是不终结的思考。它保留上下文，方便作者用更长的时间去感受，去发散，去解决。（电子笔记可重构，复原，留痕，便携）
