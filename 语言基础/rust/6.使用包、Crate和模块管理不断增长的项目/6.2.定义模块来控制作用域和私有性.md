# 📌 定义模块来控制作用域和私有性



## 📄 模块小抄

先记住四个动作：`mod` 创建或声明模块，路径定位模块中的项，`pub` 扩大可见范围，`use` 为已有路径创建较短的名称。

### 1. 从 crate 根开始构建模块树

每个 crate 都有自己的根模块。库 crate 通常从 `src/lib.rs` 开始，二进制 crate 通常从 `src/main.rs` 开始；它们是不同 crate 的根，不能理解成同一个模块树。

编译器从根文件出发，只沿着 `mod` 声明构建模块树。没有被声明的 `.rs` 文件不会因为放在 `src` 目录下就自动参与编译。

### 2. 声明模块与组织文件

模块可以直接内联，也可以拆到文件中，二者选择一种即可：

```rust
// 内联模块
mod garden {
    // 模块内容
}
```

```rust
// 文件模块：本声明通常对应 src/garden.rs 或 src/garden/mod.rs
mod garden;
```

同一个模块只能选择一种实现方式，不能同时存在两个对应文件。

任意模块都可以声明子模块。例如，在 `src/garden.rs` 中写下：

```rust
mod vegetables;
```

通常会继续查找 `src/garden/vegetables.rs` 或 `src/garden/vegetables/mod.rs`。文件只是承载模块代码的一种方式，模块本身是模块树中的节点。

### 3. 使用路径定位项

```text
crate
└── garden
    └── vegetables
        └── Asparagus
```

在隐私规则允许的前提下，`Asparagus` 可以通过完整路径引用：

```rust
crate::garden::vegetables::Asparagus
```

常用路径前缀如下：

```rust
crate::foo  // 从当前 crate 的根模块开始
self::foo   // 从当前模块开始
super::foo  // 从当前模块的父模块开始
```

`crate::` 是 Rust 提供的路径前缀，不是一个需要手动声明的普通模块名。

### 4. 私有项与公开项

模块和模块中的项默认是私有的。私有项对定义它的模块以及所有后代模块可见，但对父模块、兄弟模块和外部 crate 不可见。

```rust
mod garden {
    fn private_helper() {}

    mod vegetables {
        fn use_parent_item() {
            super::private_helper(); // 子模块可以访问祖先模块的私有项
        }
    }
}
```

使用 `pub` 可以扩大可见范围：

```rust
pub mod garden {
    pub mod vegetables {
        pub struct Asparagus;
    }
}
```

如果外部 crate 要访问 `Asparagus`，路径上的每一级模块以及 `Asparagus` 本身都必须满足可见性要求。

### 5. `use` 创建当前作用域的名称绑定

完整路径较长时，可以使用 `use`：

```rust
use crate::garden::vegetables::Asparagus;

fn plant() {
    let _plant = Asparagus;
}
```

`use` 不会移动代码，也不会自动改变隐私；它只是在当前作用域中引入一个名称。若希望把名称重新暴露为公开 API，可以使用 `pub use`。



## 📄 在模块中对相关代码进行分组

**模块**让我们可以将一个 crate 中的代码进行分组，以提高可读性与重用性。
模块同时也是隐私边界：默认情况下，模块内部的实现不会成为外部 API；只有显式标记为公开的模块或项，才能被允许的调用方使用。

### 🔖 Demo

对一个餐馆进行抽象：一个餐馆分前台（包括接待员为顾客安排座位、服务员接受点单和付款、调酒师制作饮品）、后台（厨师和烹饪人员在厨房工作、洗碗工清理餐具，以及经理处理行政事务）。使用模块的过程中对该部分代码抽象：

1. 创建 crate。`cargo new restaurant --lib` 会创建一个名为 `restaurant` 的库 crate。

2. lib.rs 中的声明：

   ```rust
   mod front_of_house {
       mod hosting {
           fn add_to_waitlist() {}
   
           fn seat_at_table() {}
       }
   
       mod serving {
           fn take_order() {}
   
           fn serve_order() {}
   
           fn take_payment() {}
       }
   }
   ```
   语法形式是：`mod 模块名 { 模块主体 }`。
   模块内可以
   
   1. 定义其它的模块，像 `hosting` 和 `serving` 。 
   2. 结构体、枚举、常量、trait

通过使用模块，可以将相关的定义分组在一起，并指出他们为什么相关，保持程序的组织性。

```rust
crate
 └── front_of_house
     ├── hosting
     │   ├── add_to_waitlist
     │   └── seat_at_table
     └── serving
         ├── take_order
         ├── serve_order
         └── take_payment
```

`src/main.rs` 和 `src/lib.rs` 分别可以作为二进制 crate、库 crate 的根文件。每个 crate 的根文件会形成该 crate 模块树的根节点。

树中的一些术语：兄弟、父子也用来描述模块嵌套结构中模块之间的关系。

1. Hosting 和 serving 是兄弟关系。
2. `front_of_house` 和 `hosting` 是父子关系。

**整个模块树都从 crate 根节点开始。** `crate::` 以这个根节点为路径起点。
```rust
// src/lib.rs
mod a {
    pub fn f() {}
}

mod b {
    pub fn g() {
        crate::a::f();      // ✅ 绝对路径：从 crate 根出发
        super::a::f();      // ✅ 相对路径：b 和 a 是兄弟，父是 crate
        a::f();             // ❌ 错误！当前模块 b 中没有 a
    }
}
```

## 📄 受限可见性

Rust 不只有“私有”和“对所有人公开”两档。`pub` 可以加路径限制:

```rust
fn local_only() {}
pub fn public_to_external_crates() {}
pub(crate) fn visible_inside_current_crate() {}
pub(super) fn visible_to_parent_module() {}
pub(in crate::service) fn visible_inside_service() {}
```

`pub(crate)` 中的是 `crate`,不是 `create`。它表示当前 crate 内部都可访问,但外部 crate 不能依赖该项。

可见性可以按“承诺范围”阅读:

```text
fn             -> 模块内部实现
pub(super)     -> 提供给直接上层组装
pub(crate)     -> crate 内部协作 API
pub            -> 外部 crate 也可依赖的公开 API
```

公开范围越大,未来修改时需要考虑的使用方越多。因此一般选择能满足当前协作的最小可见性。
