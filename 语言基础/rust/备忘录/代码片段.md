## 1. 解析文件

```rust
// 解析 mainifest
// Path 代表一个文件系统路径的不可变引用（借用）。它不拥有路径数据的所有权，只是提供一个查看路径的途径。
pub fn parse_manifest(path: &Path) -> Result<PluginManifest, Box<dyn std::error::Error>> {
    let raw = fs::read_to_string(path)?;
    let manifest = serde_json::fromstr(&raw)?;
    OK(manifest)
}
```

困惑点：

- 传参类型以及作用

  > 借用， [3.1.所有权.md](../3.所有权/3.1.所有权.md) 

- Result 封装

  >  [8.1.2用 Result 处理可恢复的错误.md](../8.错误处理/8.1.2用 Result 处理可恢复的错误.md) 
  >
  > Rust 处理可恢复错误的标准方式：
  >
  > - **定义**：`Result<T, E>` 是一个枚举，只有两个变体：
  >   - `Ok(T)`：操作成功，里面包裹着成功后的数据（这里是 `PluginManifest`）。
  >   - `Err(E)`：操作失败，里面包裹着错误信息（这里是 `Box<dyn std::error::Error>`）。

- Box\<dynamite std::error:error>

  > Rust 中动态的错误处理方式，类型擦除
  >
  > - dyn ：因为我们不知道具体会发生哪种错误（是文件读不出来？还是 JSON 格式不对？），所以我们不指定具体的错误类型，而是说“只要是实现了 `Error` 特征的**任何**错误都可以”。
  > - `Box` 是一个智能指针，它把具体的错误数据放到**堆内存**上。而 `Box` 指针本身的大小是固定的（比如 8 字节）。

- OK(manifest)

  >- **作用**：
  >  - 函数的返回类型承诺返回 `Result<PluginManifest, ...>`。
  >  - 当代码顺利执行到最后，我们手里有一个解析好的 `manifest` 对象（类型是 `PluginManifest`）。
  >  - 为了符合返回类型的要求，我们需要把它包装进 `Ok` 变体里。
  >  - `Ok(manifest)` 就等同于说：“嘿，调用者，我成功了！这是你要的数据，拿好。
