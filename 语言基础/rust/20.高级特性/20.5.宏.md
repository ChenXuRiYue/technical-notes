## Rust 中的宏

**宏**（*Macro*）指的是 Rust 中一系列的功能：使用 `macro_rules!` 的 **声明宏**（*declarative macro*），
和三种 **过程宏**（*procedural macro*）：

- 自定义 `#[derive]` 宏，用于在结构体和枚举上通过添加 `derive` 属性生成代码
- 类属性宏，定义可用于任意项的自定义属性
- 类函数宏，看起来像函数，但操作的是作为其参数传递的 token



## 为什么有了函数还需要宏？

1.  宏的 `derive` 属性，生成各种 trait 的实现。我们也在本书中一直使用 `println!` 宏和 `vec!` 宏。所有的这些宏以 **展开** 的方式来生成比你所手写出的更多的代码。

2. 宏可以在编译器编译时展开，这样提供了更强的灵活性

   >宏可以在编译器解析代码前展开，例如，宏可以在一个给定类型上实现 trait。而函数则不行，因为函数是在运行时被调用，而 trait 需要在编译时实现。

## [用 `macro_rules!` 编写用于通用元编程的声明宏](https://kaisery.github.io/trpl-zh-cn/ch20-05-macros.html#用-macro_rules-编写用于通用元编程的声明宏)

**声明宏**允许我们编写一些类似 Rust `match` 表达式的代码。

几个关键的特性：

- 降至和包含相关代码的模式进行比较
- 匹配成功则替换传递给宏的代码
- 发生于编译时

**例子**

```rust
let v: Vec<u32> = vec![1, 2, 3];
```

宏定义：

```rust
#[macro_export]
macro_rules! vec {
    ( $( $x:expr ),* ) => {
        {
            let mut temp_vec = Vec::new();
            $(
                temp_vec.push($x);
            )*
            temp_vec
        }
    };
}
```

它将会替换为如下形式：

```rust
{
    let mut temp_vec = Vec::new();
    temp_vec.push(1);
    temp_vec.push(2);
    temp_vec.push(3);
    temp_vec
}
```

## [用于从属性生成代码的过程宏](https://kaisery.github.io/trpl-zh-cn/ch20-05-macros.html#用于从属性生成代码的过程宏)

**过程宏**
过程宏接收 Rust 代码作为输入，在这些代码上进行操作，然后产生另一些代码作为输出，而非像声明式宏那昂匹配对应模式然后以另一部分代码替换当前代码。



一些令人困惑的点：

**根据官方文档：**

创建过程宏时，其定义必须驻留在它们自己的具有特殊 crate 类型的 crate 中。这么做出于一些复杂的技术原因，将来我们希望能够消除这些限制。在示例 20-36 中展示了如何定义过程宏，其中 `some_attribute` 是一个使用特定宏变体的占位符。

也就是说，它定义上会比较麻烦：

- 新建一个 一个 特殊的crate 。将使用它的代码和定义它的代码分开：

  项目结构大概下边这样的：

  ```
  my_project/ (你的主项目)
  my_macros/ (一个专门用来存放过程宏的独立 crate)
  ```

  ```toml
  # my_macros/Cargo.toml
  [lib] # 指定了他的编译顺序
  proc-macro = true # 这行配置将这个库标记为“过程宏 crate”
  ```

- ❗️ 在后续版本。该语法或许会得到优化

### **一个简单的例子：**

- 为一个结构体增加 hello_macro() 方法

  ##### 过程宏定义：

  **Cargo.toml**

  ```rust
  # hello_macro_derive/Cargo.toml
  [package]
  name = "hello_macro_derive"
  version = "0.1.0"
  edition = "2021"
  
  [lib]
  proc-macro = true # <--- 关键配置：声明这是一个过程宏 crate
  
  [dependencies]
  # syn: 用于解析 Rust 代码成语法树
  syn = "2.0"
  # quote: 用于将语法树转换回 Rust 代码
  quote = "1.0"
  # proc-macro2: 处理底层的 token 流
  proc-macro2 = "1.0"
  ```

  **宏逻辑**

  ```rust
  use proc_macro::TokenStream;
  use quote::quote;
  use syn;
  
  // 定义宏函数，使用 #[proc_macro_derive] 属性标记
  #[proc_macro_derive(HelloMacro)]
  pub fn hello_macro_derive(input: TokenStream) -> TokenStream {
      // 1. 解析：将输入的 TokenStream 解析为 Rust 的语法树 (AST)
      // 这里我们只关心它是一个结构体或枚举，所以用 syn::parse_macro_input!
      let ast = syn::parse(input).unwrap();
  
      // 2. 处理：调用我们自定义的函数来构建代码
      impl_hello_macro(&ast)
  }
  
  // 这是一个普通的 Rust 函数，负责生成代码逻辑
  fn impl_hello_macro(ast: &syn::DeriveInput) -> TokenStream {
      // 获取结构体或枚举的名字 (例如 struct Pancakes，这里获取 "Pancakes")
      let name = &ast.ident;
  
      // 3. 生成：使用 quote! 宏来拼凑代码
      // 注意 #name 会被变量 name 的值替换
      let gen = quote! {
          impl HelloMacro for #name {
              fn hello_macro() {
                  // stringify! 将标识符直接转换为字符串字面量
                  println!("Hello, Macro! My name is {}!", stringify!(#name));
              }
          }
      };
  
      // 将生成的代码转换回 TokenStream 返回给编译器
      gen.into()
  }
  ```

  ##### 主库 crate

  ```toml
  # hello_macro/Cargo.toml
  [package]
  name = "hello_macro"
  version = "0.1.0"
  edition = "2021"
  
  [dependencies]
  # 依赖刚才写的那个宏 crate
  hello_macro_derive = { path = "../hello_macro_derive" }
  ```

  库代码

  ```rust
  // 1. 定义一个 Trait
  pub trait HelloMacro {
      fn hello_macro();
  }
  
  // 2. 定义一个结构体，并使用我们刚才写的宏
  use hello_macro_derive::HelloMacro;
  
  #[derive(HelloMacro)] // <--- 魔法发生在这里
  pub struct Pancakes;



## [类属性宏](https://kaisery.github.io/trpl-zh-cn/ch20-05-macros.html#类属性宏)

类属性宏允许为对象创建新的属性。
经典示例：

自动日志记录器：

```rust
use proc_macro::TokenStream;
use quote::quote;
use syn::{parse_macro_input, ItemFn};

// 使用 #[proc_macro_attribute] 标记这个函数是一个属性宏
#[proc_macro_attribute]
pub fn log_execution(_attr: TokenStream, item: TokenStream) -> TokenStream {
    // 1. 解析：将输入的 item 解析为一个函数 (ItemFn)
    let input_fn = parse_macro_input!(item as ItemFn);
    
    // 提取函数名
    let fn_name = &input_fn.sig.ident;
    
    // 提取函数体
    let fn_body = &input_fn.block;

    // 2. 生成：构建新的代码
    // 我们保留原函数名，但在函数体外面包了一层打印逻辑
    let expanded = quote! {
        fn #fn_name() {
            println!(">>> 正在进入函数: {}", stringify!(#fn_name));
            
            // 执行原始函数体
            #fn_body
            
            println!("<<< 函数执行完毕: {}", stringify!(#fn_name));
        }
    };

    // 3. 返回
    TokenStream::from(expanded)
}
```

**使用：**

定义宏：

```rust

## [类函数宏](https://kaisery.github.io/trpl-zh-cn/ch20-05-macros.html#类属性宏)

类函数（Function-like）宏的定义看起来像函数调用的宏。类似于 `macro_rules!`，它们比函数更灵活；例如，可以接受未知数量的参数。

**生成自动化测试代码的例子：**

定义：

```rust
use proc_macro::TokenStream;
use quote::quote;
use syn::{parse_macro_input, Ident};

#[proc_macro]
pub fn auto_test(input: TokenStream) -> TokenStream {
    // 1. 解析：获取函数名 (例如: my_func)
    let name = parse_macro_input!(input as Ident);
    
    // 2. 生成：拼接出 test_my_func 测试函数
    let test_name = Ident::new(&format!("test_{}", name), name.span());

    let expanded = quote! {
        #[cfg(test)]
        mod tests {
            use super::*;
            
            #[test]
            fn #test_name() {
                // 自动生成：调用函数并断言它不 panic (这里简单演示)
                // 实际落地中，你可以生成更复杂的断言逻辑
                let _result = #name(); 
                // println!("Auto-test passed for {}", stringify!(#name));
            }
        }
    };

    TokenStream::from(expanded)
}
```

使用：

```rust
use my_macros::auto_test;

fn add(a: i32, b: i32) -> i32 {
    a + b
}

// 使用宏：自动生成 test_add 函数
// 编译后，代码中会多出一个 #[test] fn test_add() { ... }
auto_test!(add);

fn main() {
    println!("Running...");
}
```



## 简单汇总

| 特性        | 声明宏                           | 过程宏                                                       |
| ----------- | -------------------------------- | ------------------------------------------------------------ |
| 关键字/属性 | `macro_rules!`                   | `#[proc_macro]` `#[proc_macro_derive]` `#[proc_macro_attribute]` |
| 本质        | 模式匹配 + 文本替换              | 编译时运行的 Rust 函数                                       |
| 输入/输出   | 接收代码片段，直接替换           | 接收 `TokenStream`，返回 `TokenStream`                       |
| 复杂度      | ⭐ (简单，适合日常)               | ⭐⭐⭐ (复杂，适合库开发)                                       |
| 常见用途    | 简化重复逻辑、`println!`、`vec!` | 自动生成代码 (`#[derive]`)、DSL、框架路由                    |



## 反复困惑

1. TokenStream 是什么？

   >编译器用一个 Token 流的模型查看源代码：
   >例如：println!("Hello, {}", name);
   >
   >1. **标识符**: `println`
   >2. **标点**: `!`
   >3. **标点**: `(`
   >4. **字面量**: `"Hello, {}"`
   >5. **标点**: `,`
   >6. **标识符**: `name`
   >7. **标点**: `)`
   >8. **标点**: `;`

   **TokenStream** 就是这些 Token 组成的**流（序列）**

2. 语法关键词有什么？为什么要这样命名这些关键词？
