# 📌 TypeScript 解构赋值

基础谈资：解构赋值用于从对象或数组中批量取得值，并把这些值绑定到局部变量。对象解构按属性名匹配，数组解构按位置匹配。该语法来自 JavaScript，TypeScript 在它之上增加类型检查和类型推断。

## 📄 对象解构

### 🔖 对象解构：按照属性名匹配

```ts
const user = {
  name: "小明",
  age: 18,
};

const { name, age } = user;
```

等价于：

```ts
const name = user.name;
const age = user.age;
```

属性的书写顺序不影响结果：

```ts
const { age, name } = user;
```

对象解构左侧的 `{ name, age }` 是一个**解构模式**，不是在创建新的对象字面量。

## 📄 数组解构

### 🔖 数组解构：按照位置匹配

```ts
const values = ["read", 320];
const [mode, width] = values;
```

等价于：

```ts
const mode = values[0];
const width = values[1];
```

React `useState` 经常使用数组解构：

```tsx
const [mode, setMode] = useState("read");
```

这里的变量名可以修改，因为数组解构只看位置：

```tsx
const [value, updateValue] = useState("read");
```

### 🔖 对象解构与数组解构的核心区别

```text
const { name } = object;
  -> 对象解构
  -> 按属性名 name 匹配

const [value] = array;
  -> 数组解构
  -> 按索引位置匹配
```

## 📄 重命名、默认值与剩余属性

### 🔖 解构时重命名

```ts
const { name: userName } = user;
```

这里表示：

```text
读取 user.name
  -> 创建局部变量 userName
```

等价于：

```ts
const userName = user.name;
```

需要注意：`name: userName` 不是类型注解，也不是把 `userName` 写回对象。

### 🔖 解构时提供默认值

```ts
const { role = "guest" } = user;
```

只有属性值为 `undefined` 时才使用默认值：

```ts
const source = { role: undefined };
const { role = "guest" } = source;
// role 为 "guest"
```

如果值为 `null`，默认值不会生效。

### 🔖 剩余属性

```ts
const { name, ...rest } = user;
```

结果可以理解为：

```text
name -> user.name
rest -> 除 name 之外的其余可枚举自有属性组成的新对象
```

数组也支持剩余元素：

```ts
const [first, ...others] = [1, 2, 3];
// first = 1
// others = [2, 3]
```

### 🔖 函数参数解构

```ts
interface UserOptions {
  name: string;
  enabled?: boolean;
}

function printUser({ name, enabled = true }: UserOptions) {
  console.log(name, enabled);
}
```

函数收到的是一个 `UserOptions` 对象，进入函数时立即取出 `name` 和 `enabled`。

## 📄 解构与 TypeScript 类型检查

### 🔖 TypeScript 如何参与

解构语法由 JavaScript 提供，TypeScript 主要补充以下能力：

1. 检查被解构的属性是否存在。
2. 根据源对象推断局部变量类型。
3. 检查默认值是否与属性类型兼容。
4. 检查函数参数的解构结构。

```ts
interface User {
  name: string;
  age: number;
}

declare const user: User;

const { name, age } = user;
// name 被推断为 string
// age 被推断为 number

const { missing } = user;
// 类型错误：User 上不存在 missing
```
