# 局域网常识

##### Q1：局域网内设备被分配了私有 IP（如 `192.168.x.x`），为什么能访问公网页面？

三层机制串联：

- **接入层（DHCP）**：设备接入网络时自动向 DHCP 服务器请求 IP，同时获得子网掩码、默认网关和 DNS 服务器地址，无需手动配置
- **转发层（交换机 + 路由器 + NAT）**：交换机基于 MAC 地址在局域网内转发；路由器充当网关，通过 NAT 将私有 IP 映射到公网 IP
- **通信层（DNS + TCP/IP）**：DNS 解析域名获取目标 IP，TCP/IP 建立连接，ARP 解析 MAC 地址，以太网处理帧传输

> 🌱 核心：DHCP 分配私有 IP → NAT 映射到公网 → TCP/IP 通信。即使没有固定 IP，设备也能正常上网。

---

##### Q2：那 IPv4 地址够用吗？我设备的 IP 是什么，稳定吗？

IPv4 不够，靠 NAT 补：一个公网 IP 挂几十台设备，内网用 `192.168.x.x`。你家路由器干的就是这件事——外面看到一个公网 IP，里面手机电脑平板共用。

你实际上有两个 IP：路由器分的**内网 IP**（`192.168.x.x`，换 Wi-Fi 就变）和路由器在运营商那拿到的**公网 IP**（所有设备共享，路由器重启可能换）。手机用蜂窝数据时走运营商分配的另一个 IP，更不稳定。

大多数通信不需要知道对方 IP——**你主动发起请求，服务器顺着连接回包**，NAT 表记录"这个请求是谁发的"，回包自然能找到你。只有别人要主动连你时（P2P、自建服务器）才需要固定公网 IP。

---

##### Q3：既然 IP 不固定，服务器怎么找到我？

大多数情况下不需要知道你的 IP。通信是**你主动发起的**——你发请求，服务器顺着连接回包，NAT 表记录"这个请求是谁发的"，回包自然能找到你。

只有别人要**主动连你**时（P2P、自建服务器）才需要固定公网 IP。这也是为什么普通人不需要固定 IP。
