# 🎮 Agent 游戏裁判规则

> **分层加载**：默认只读「第一层」。仅当路由表的触发条件命中时，才打开对应深层文档，节省 token。适用于任何 AI。

---

## 🔥 第一层 · 单任务审计（默认只读这里）

99% 的场景：完成一个任务，记一次分。全程只读 `state.json`，不读大表。

### 步骤

1. 读 `成就系统/数据/state.json` 拿当前状态（含缓存阈值、计数器）
2. 读取任务条目末尾用户自标的档位（数字 `1-5` 或 emoji，映射 1⚡/2🟢/3🟡/4🔴/5💀）→ 基础经验、金币（金币 = 经验 × 0.5 向下取整）。未标注则 git 兜底按最低档 ⚡
   - **摇奖**：读任务所在爆率分区（🎰头奖30% / 🎯有趣15% / 🎲普通5%，待定区或无分区按 5%），算 `roll = hash(审计时间戳 + 任务名) % 100`，`roll < 爆率` → 暴击，基础经验 ×2
3. 追加 `成就系统/数据/exp.jsonl`（经验记录）和 `成就系统/数据/coin.jsonl`（金币收入记录）
4. 更新 `state.json`：
   - `exp_total` / `coin_balance` / 各 `*_exp` 累加
   - `combo`（与上次完成同段连续则 +1）
   - `tasks_completed` +1
   - `daily_streak`：若 `last_active_date` 是昨天则 +1，是今天则不变，否则归 1；更新 `last_active_date`
   - `active_days_total`：若本次是新的一天则 +1
   - 必要时刷新各 `*_next` 赛季阈值
5. 查成就：对照明置/隐藏条件（仅用 state + 本次 ts，零额外读取），命中且不在 `achievements[]` 则解锁、发奖、记入
6. 查境界试炼：当前 `realm` 的下一重条件是否达成（火种→微光：`active_days_total`≥180 且 `tasks_completed`≥100 且连续 6 月无断更）。达成则提示「已具备突破条件」，**不自动进阶**（见第二层 · 境界突破）
7. 更新 `任务面板.md` 顶栏
8. 将已完成任务从原爆率分区**移动**到 `🏆 已征服` 区（删除原位置条目，追加到已征服区末尾），格式：`档位emoji 任务名 `爆率% 掷N` `+Nexp +Ncoin``，暴击时末尾加 `🎰`。骰子规则：`掷N < 爆率%` → 🎰暴击(exp×2)。例：``🟢 Get 和 Post 对比 `🎲5% 掷12` `+3exp +1coin` ``
9. **输出结算卡**（见下）

### 结算卡（每次审计必出，这是核心反馈）

把记账变成开箱仪式。格式：

```
┌─ 结算 ─────────────────
│ ✅ 任务名  [档位] +N经验 +N金币
│ 🎰 暴击！经验 ×2  (爆率N% 掷出N)   ← 暴击才显示
│ 🔥 连击 ×N  (+N)
│ 🎉 解锁成就：成就名 +N    ← 有才显示
│ ──────────────────────
│ 🔥 境界 火种 ｜ 经验 A→B · 金币 C
│ ⛰️ 月 ⛺扎营 ▱▱▱▱ B/阈值
│ 🌊 季 🏖️港湾 ▱▱▱▱ B/阈值
│ 🌍 年 🌱苔原 ▱▱▱▱ B/阈值
└────────────────────────
```

暴击 / 跨赛季档 / 完成挑战 / 具备境界突破条件时，额外加一行高亮提示。

### 评分表

档位由用户勾选完成时自标（末尾敲数字 1-5），Agent 直接读取并转 emoji 归档，不推断。

| 标 | 档位 | 名字 | 时间 | 经验 | 金币 |
|----|------|------|------|------|------|
| 1 | ⚡ | 顺手 | < 15min | 1 | 0 |
| 2 | 🟢 | 轻松 | 15min - 1h | 3 | 1 |
| 3 | 🟡 | 投入 | 1h - 4h | 7 | 3 |
| 4 | 🔴 | 硬仗 | 4h - 1天 | 15 | 7 |
| 5 | 💀 | 鏖战 | > 1天 | 25 | 12 |

- **一个任务** = 任务面板 中一个顶层 `- [x]` 条目；缩进子任务不单独计分，整条完成记一次。
- **未标档位** → git 兜底，按最低档 ⚡。
- **连击**：每记一个任务 combo +1（2 连 +1、≥5 连 +2）；7 天无完成归零，不扣分。

### 路由表 — 何时进入第二层

只有命中以下条件才打开对应文档，否则**不读**：

| 触发条件 | 打开 |
|---------|------|
| 具备境界突破条件（步骤 6 命中） | 下方 · 境界突破 |
| 某 `*_exp` ≥ 对应 `*_next` | GAMIFICATION.md「赛季」→ 盖章，刷新 `*_next` |
| 任务带 `[项目名]` 标签 | GAMIFICATION.md「目标分类」→ 里程碑/完结奖励 |
| 「📥 待定区」有未归类任务 | GAMIFICATION.md「爆率系统」→ **派 subagent** 按 5 维评估，移入对应爆率分区（见下「标爆率」）|
| 月初（无当月 `month_challenge`） | GAMIFICATION.md「月度挑战」→ 设定本月挑战 |
| 用户说"兑换/花金币" | 下方 · 金币兑换 |
| 用户说"盘点/对账" | 下方 · 全局盘点 |
| 月/季/年末 或 用户说"回顾" | 下方 · 赛季回顾 |

> 成就判定已内联在第一层步骤 5（明置看 state 字段、隐藏看本次 ts），无需打开 GAMIFICATION，条件表仅在不确定奖励数值时查阅。
> `state.json` 缓存了各 `*_next`，判断是否跨赛季档只需一次比较，无需读大表。境界靠里程碑试炼，不由积分驱动。

### 💰 模型建议

单任务审计是纯机械的 git→分数映射，**用便宜模型即可**（DeepSeek V3 / Qwen / Haiku），无需推理模型。"教授"角色（知识引导、写笔记）才需要强模型。两者可分离。

### 💡 教授荐题（偶尔推荐，不打扰）

审计时（已读 git），若命中下列信号才推 **1 条**，无信号则沉默；同一天最多 1 条。

| 信号 | 题型 |
|------|------|
| 刚完成的事有自然延伸 | 延伸题（深化）|
| 近期 git 反复碰同类痛点 | 造物题（做成 SKILL/笔记 → 甜点区 🎰）|
| 某领域笔记明显缺口 | 补全题（体系完整）|
| 某 Boss 子任务/待办搁置很久 | 唤醒题 |

- 优先推 体系/造物/痛点 方向，不推纯消费型学习
- 形式：结算卡后附 1 行，含预估档位+爆率，例：
  `💡 荐题：把本周 3 次 checkStyle 的坑做成 SKILL？[🔴硬仗 · 预估🎰]`
- 荐题写入面板「🎁 随机推荐」区（不入爆率分区、不计待办压力）；用户接受则移入对应爆率分区，忽略可丢进 🗑️ 垃圾桶

### ⏰ 触发时机

**审计**：用户准备执行 `git commit` 前，自动触发一次审计。流程：
1. 执行上方步骤 1-7（读 state → 推断 → 写账本 → 结算卡）
2. 结算卡输出后，再执行 commit

**标爆率**（每次读取 `任务面板.md` 时自动检测）：发现「📥 待定区」有任务 → **派一个 subagent** 批量归类，不阻塞当前对话：
1. 子代理读 GAMIFICATION.md「爆率系统」，按 5 维模型逐条评估（偏保守，拿不准下调一档；🌐体系＋任一其他维度→🎰，单命中体系→🎲，其余按命中数 0-1→🎲 / 2-3→🎯 / ≥4→🎰）
2. 把每条**移动**到对应爆率分区（🎰/🎯/🎲）、清空待定区；任务行不加任何符号（爆率由分区表达），子任务跟随父任务
3. **只改 `任务面板.md`**，不碰账本 / state / 顶栏；完成后回报归类清单

**回顾**（按需，第二层）：

触发逻辑统一为**跨周期检测**——commit 前审计时，对比 state 中记录的周期与当前日期：

| 检测 | 条件 | 动作 |
|------|------|------|
| 周回顾 | 今天是周日 且 本周未生成 | 生成本周回顾 |
| 月回顾 | 当前月 ≠ `state.season.month` | 生成上月回顾，month_exp 归零，切换 month |
| 季回顾 | 当前季 ≠ `state.season.quarter` | 生成上季回顾，quarter_exp 归零，切换 quarter |
| 年回顾 | 当前年 ≠ `state.season.year` | 生成上年回顾，year_exp 归零，切换 year |

即：**跨入新周期的第一次 commit，自动补生成上一周期的回顾**。不需要预判"最后一次"。

回顾生成后归档到 `成就系统/回顾/` 对应子目录：

| 回顾 | 归档路径 | 文件名 |
|------|---------|--------|
| 周回顾 | `成就系统/回顾/周/` | `YYYY-WNN.md` |
| 月回顾 | `成就系统/回顾/月/` | `YYYY-MM.md` |
| 季回顾 | `成就系统/回顾/季/` | `YYYY-QN.md` |
| 年回顾 | `成就系统/回顾/年/` | `YYYY.md` |

## 📂 第二层 · 按需场景

### 境界突破（步骤 6 命中试炼）

1. 提示「你已具备突破【下一境界】的条件」，**不自动进阶**
2. 由用户亲自确认进阶，并写一句留念
3. 将留念写入 `成就系统/回顾/境界/X.md`，更新 `state.realm`
4. 触发成就 🌟突破！；结算卡高亮
> 境界条件见 GAMIFICATION.md「生涯境界」。恒星、银河当前留白，走到星火再定义。

### 金币兑换（用户说"兑换/花金币"）

1. 核对 `coin_balance` ≥ 奖励价格（见 GAMIFICATION.md「奖励商店」）
2. 追加一条 `redeem` 到 `成就系统/数据/coin.jsonl`：`{"ts":...,"type":"redeem","desc":"奖励名","coin":-N}`
3. `coin_balance` 扣减，更新 任务面板 顶栏
4. 经验、等级、赛季**不受影响**（金币是独立的犒劳货币）

### 全局盘点（用户说"盘点/对账"）

一段时间未记录，需追溯补录：
1. 读 `成就系统/数据/exp.jsonl` 最后一条时间戳 = 上次截止点
2. `git log --since="上次时间"` 查看期间提交
3. 对比 `任务面板.md` 已完成区 vs ledger，找出未入账任务
4. 逐条按 git 推断档位，批量写入 ledger，更新 state 和 任务面板 顶栏
5. 输出盘点摘要

### 垃圾桶（用户说"丢弃/不做/扔了"）

不是所有任务都值得做：
1. 把该条从 `任务面板.md` **剪切**到 `成就系统/垃圾桶.md`，可附一句弃因
2. **不计分、不惩罚**，不碰账本 / state / 连击
3. 子任务随父任务一起移走

### 赛季回顾（周期末 / 用户说"回顾"）

按 `成就系统/回顾/` 下的范式生成，归档到 `回顾/月`、`回顾/季`、`回顾/年`。范式见 `成就系统/README.md`。

## 原则

- **不追问用户**，从 git 历史自主判断；信息不足按最低档 ⚡ 记，宁少不多
- ledger 只追加不修改，state 是其投影
