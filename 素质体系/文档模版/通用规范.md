## 模版选型

新建笔记时，根据内容类型选择对应模版：

| 内容类型 | 模版 | 特征 |
| -------- | ---- | ---- |
| 配置 / 工具 / 命令行速查 | 工具书模版 | `## 🔖` 分节 + `⚙️` 子节 + `🪏 经验` 表格 |
| 知识学习 / 原理理解 | 技术学习模版 | 基础谈资 + topic 分节 + 生长思考 + 反复绊脚 |

判断依据：`个性配置/`、`开发环境/` 目录下大概率是工具书模版；`专业基础/`、`后端工程/` 等偏知识理解的目录用技术学习模版。如有疑问，先看同目录已有文件的风格。

## 创建笔记规则

- **先看后写**：动笔前必须先读对应模版文件 + 同目录已有文件，继承其 emoji、分节、排版风格，不要自己发明
- **只生成文档骨架**（标题 + 模版要求的空分节），不替用户决定主题、结构、主要内容
- 调研结果追加到文档末尾的 `## 调研` 区，仅记录相关经验和信息
- 文档内容由用户自己填充
