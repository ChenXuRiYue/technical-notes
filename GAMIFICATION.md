## 积分体系

### 任务难度

| 标签 | 基础分 | 说明 |
| ---- | ---- | ---- |
| 🟢 简单 | 2 | 查个文档就能解决 |
| 🟡 中等 | 5 | 需要调研或学习，1-2 小时 |
| 🔴 困难 | 10 | 涉及深度思考或架构设计 |
| 💀 地狱 | 20 | 跨领域、高复杂度、长期攻坚 |

### 计算公式

```
任务得分 = 难度分
```

## 传奇 Boss

普通任务刷经验，传奇 Boss 定义游戏本身。

传奇 Boss 不参与五维打分。它不是「待办」，它是「命运」。它贯穿所有体系，跨越所有阶段，甚至反过来改变积分系统本身。

### 当前 Boss

| Boss | 状态 | 阶段 | 说明 |
| ---- | ---- | ---- | ---- |
| 🐉 GitTributary | 🟡 进行中 | 调研 → 实验 → 开发 → 发布 | 插件化笔记增强桌面应用，贯穿一生的基础设施 |

### Boss 进度追踪

Boss 不打分，按阶段追踪：

```
[███░░░░░░░] 30% — 调研完成，进入实验阶段
```

每个阶段完成时，触发特殊成就。

### Boss 与任务系统的关系

- Boss 下的子任务（如「完成 Lab 1.1」）可以作为普通任务打分
- 但 Boss 本身是凌驾于任务之上的存在
- 完成一个传奇 Boss = 直接通关

## 扩展预留

以下机制待后续迭代：

- **赛季系统**：按月/季度重置连击和部分成就，保持新鲜感
- **挑战任务**：限时任务，完成额外加分
- **支线任务**：非技术类任务（如健身、阅读），全面发展
- **排行榜**：如果未来有多人协作，可引入团队排行
- **技能树**：按技术方向（后端、前端、AI、运维）分支，点亮技能节点
