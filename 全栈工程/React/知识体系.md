# 📌 React 知识体系

React 的知识体系可以切成一棵树。读项目代码、读官方文档时，先定位「这块属于哪个模块」，再往下钻细节，比平铺记一堆 API 高效。

```text
React 知识体系
├── 核心概念       元素/组件/渲染/虚拟DOM/reconciliation/JSX
├── 组件模型       函数组件 vs 类组件/props/children/组合 vs 继承
├── 状态与副作用
│   ├── Hooks        useState/useEffect/useRef/useContext/useReducer/useMemo/useCallback
│   └── 状态管理      Context/外部状态库
├── 事件与交互       合成事件/受控非受控组件/表单
├── 性能优化         memo/useMemo/useCallback/key/虚拟化
├── 生态             Router/SSR/测试
└── 哲学             单向数据流/声明式/UI = f(state)
```

## 📄 模块定位

每个模块回答一个问题，知道问题就知道该往哪个模块挂笔记：

| 模块 | 问题 | 目录 |
| --- | --- | --- |
| 核心概念 | React 怎么把数据变成页面？ | `核心概念/` |
| 组件模型 | 怎么把 UI 拆成可复用单元？ | `组件模型/` |
| 状态与副作用 | 组件要记住什么、要对外界做什么？ | `状态与副作用/` |
| 事件与交互 | 用户输入怎么进系统？ | （待建） |
| 性能优化 | 渲染太慢怎么治？ | `性能优化/` |
| 生态 | 路由/SSR/测试怎么接？ | （待建） |
| 哲学 | React 的设计心法是什么？ | `React哲学.md` |
